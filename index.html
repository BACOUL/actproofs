Automation created a responsibility gap

AI systems and automated workflows already delete data, send messages, trigger decisions, and impact users and systems.

Who authorized this action?

Logs show what happened. Permissions show what was possible. Policies describe intent.

None provide proof that an action was explicitly authorized before it occurred.

What regulations actually require

Regulation does not mandate tools. It mandates the ability to demonstrate control, prove authorization, and assign responsibility.

  • Prove authorization
  • Audit automated actions
  • Demonstrate accountability
  • Assign responsibility after the fact

Without proof of authorization, responsibility is assumed — not demonstrated.

What ActProofs is

ActProofs is a standard and API that produces a cryptographic, portable, independently verifiable receipt proving that a specific automated action was explicitly authorized by an identified actor, in a defined context, at a given point in time.

Actor → Authorization → ActProof → Automated Action

ActProofs does not manage identity, execute actions, or decide whether an action should occur.

It only records authorization.

ActProofs provides technical proof mechanisms only. It does not constitute legal advice or certification of compliance.

How it fits into automation

  1. An automated system requests an authorization proof
  2. ActProofs issues a verifiable receipt
  3. The system proceeds and keeps the proof

ActProofs does not slow automation. It makes automation accountable.
