A log entry is not proof

Execution logs are mutable. An ActProof is a signed artifact that proves who authorized what, independent of any database.

Standard vs. Service

ActProofs is an open standard with an optional managed issuance service.

The Standard (ActSpec)

Free & Open Source

  • Free to implement and use.
  • Verification is always free, offline, and stateless.
  • No vendor lock-in.
  • Reference implementations on GitHub.

The Service (ActProofs.io)

Optional Managed Issuance

  • We operate a governed signing authority.
  • Use our API to sign proofs without key management.
  • Audit trails and compliance dashboards.
  • Note: Verification remains possible even if this service is offline.

What ActProofs is (and is not)

What you get

A portable JSON file containing the authorization manifest and a cryptographic signature.

Why it‚Äôs safe

Stateless and deterministic. The proof travels with the data or stays in your audit logs.

What it is NOT

It is NOT execution. ActProofs generates the receipt of authorization; it does not trigger or guarantee the action's success.

Boundaries & Trust Model

‚ö†Ô∏è Crucial Distinction: Authorization ‚â† Execution

ActProofs certifies that an intent was authorized by a valid key at a specific time. It does not guarantee that the external system executed the action successfully, nor does it enforce the action. It bridges the gap between Identity (IAM) and Audit, but does not replace execution logs.
