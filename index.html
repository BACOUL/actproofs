A log entry is not proof

Execution logs are centralized and lack independent verifiability. An ActProof is a signed, portable artifact: it proves who authorized what, when, and in what context.

ActProofs doesn't execute or enforce. It fills the gap between access control and audit by certifying one fact: explicit authorization of an intent.

What ActProofs is (and is not)

What you get

A proof file containing the authorization manifest (.actproof.json).

A signed receipt from a stateless engine (no persistent database needed).

Independent verification that the action parameters match the signed intent.

Why it‚Äôs safe

Stateless: the system doesn't store your authorization data.

Portable: the proof travels with the data or remains in audit trails.

Deterministic: provides a consistent output for every specific authorization event.

What it‚Äôs not

Not Execution: ActProofs does not trigger or run the action.

Not IAM: It doesn't manage users, roles, or permissions policies.

Not Logging: It is not a streaming log service or a storage layer.

How It Works

Three steps: Define Context ‚Üí Sign Authorization ‚Üí Verify the Receipt

1) Define Context

Map the actor, the intended action, and the specific parameters into a standard JSON manifest.

2) Sign Authorization

The system generates a cryptographic signature of the manifest, producing a portable .actproof.json file.

3) Verify the Receipt

Any auditor or system can verify the signature against the original manifest to prove the action was authorized.
